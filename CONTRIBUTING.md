# Contributing to Urban Studio

Thank you for your interest in contributing to Urban Studio. We welcome your contributions to help make this project even better. Please take a moment to review these guidelines to ensure a smooth and effective contribution process.

## How to Contribute

To contribute to Urban Studio, follow these steps:

1. Fork the repository to your GitHub account.

2. Clone the forked repository to your local machine:

   ```bash
   git clone https://github.com/yourusername/urban-studio.git
   ```

3. Create a new branch for your contribution:

   ```bash
   git checkout -b feature/your-feature-name
   ```

4. Make your changes, improvements, or fixes.

5. Ensure that your code adheres to the project's coding standards and conventions.

6. Test your changes thoroughly to avoid introducing new issues.

7. Commit your changes with a clear and concise commit message:

   ```bash
   git commit -m "Add feature: your feature description"
   ```

8. Push your changes to your fork on GitHub:

   ```bash
   git push origin feature/your-feature-name
   ```

9. Create a Pull Request (PR) from your fork to the main project repository. Be sure to provide a detailed description of your changes and explain why they should be merged.

## Coding Standards

- Follow the PEP 8 style guide for Python code.
- Maintain code readability and consistency.
- Write clear and concise comments and documentation for new features or code changes.

## Issue Tracking

You can also contribute by reporting issues, suggesting improvements, or requesting new features. Use the project's [issue tracker](https://github.com/h-tr/urban-studio/issues) to create and discuss issues.

## Code of Conduct

Please note that this project follows an inclusive and welcoming community. We expect all contributors to adhere to the [Code of Conduct](CODE_OF_CONDUCT.md). Any violations will not be tolerated.
